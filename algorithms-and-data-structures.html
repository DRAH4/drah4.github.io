Algorithms & Data Structures Artifact

Artifact Description

For this enhancement, I focused on the algorithms and data structure component of my Full Stack Ticketing System. This artifact includes the sorting and filtering logic used on the ticket list page, which I built during this milestone as the project shifted from a static layout to a functional application. The original version had hard-coded rows and no real logic for organizing or processing ticket data.

Why I Chose This Artifact

I chose this artifact for my ePortfolio because it demonstrates practical algorithmic skills in a real application. It shows how I designed solutions that sort tickets by date, apply multiple filters at the same time, and handle actual data instead of static HTML. These improvements made the ticket list fully interactive and reflect my ability to use algorithms to solve real-world problems.

Key Enhancements

The main enhancements implemented for this milestone include:

  • Building a dynamic renderTickets(list) function that generates table rows from an array of ticket objects.
  • Implementing quick filters such as “Oldest Tickets,” “High Priority,” and “Open Tickets.”
  • Adding dropdown filters for type, status, owner, priority, and assignee that can all work together.
  • Developing a unified applyFilters() function that evaluates all filter conditions at once.
  • Ensuring that clearing one filter does not reset the others by improving the filtering logic.

Course Outcomes Demonstrated

This enhancement supports the outcome related to designing and evaluating computing solutions using algorithmic principles. I had to think about the structure of the filtering logic, how to prevent conflicts between filters, and how to manage trade-offs between readability and flexibility. The result is a clean and maintainable approach for processing ticket data.

Reflection on the Process

Initially, each filter worked independently, which caused issues—clearing one filter would reset the list and ignore the others. Designing a single combined filtering function solved this and helped me better understand how to handle multiple conditions with arrays. It also taught me the importance of data flow and when to trigger UI updates. This enhancement aligned well with my planned outcomes, and I didn’t need to change my outcome coverage plan because the work naturally fit the goals I set for this milestone.
