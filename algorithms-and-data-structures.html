Algorithms and Data Structures Artifact

Artifact Description

This artifact is the ticket list and filtering logic inside my Full Stack Ticketing System. The original version of the ticket list was just static HTML rows. There was no real data handling, and the page did not react to user input beyond the basic select dropdowns.

Why I Chose This Artifact

I chose this piece for the algorithms and data structures category because it is where I started treating the tickets as real data instead of just static markup. It let me show that I can take an array of ticket objects, sort and filter them, and keep the UI in sync as the user interacts with the page.

Key Enhancements

For this category, I focused on the logic that works with the ticket data:

  • Created a renderTickets(list) function that takes an array of ticket objects and builds the table rows dynamically.
  • Added “Oldest Tickets,” “High Priority,” and “Open Tickets” quick filter buttons that sort or filter the tickets based on created date, priority, or status.
  • Added dropdown filters for type, status, owner, priority, and assignee that can all work together at the same time.
  • Wrote a central applyFilters() function that checks each active filter and returns only the tickets that match all of them.
  • Updated the code so ticket data comes from storage or the backend and is passed through the same render and filter logic.

Course Outcomes Demonstrated

These enhancements support the outcome about designing and evaluating computing solutions that solve a problem using algorithmic principles, while managing trade offs. I had to think about how to structure the filters so they did not fight each other, and how to keep the logic simple enough to maintain but flexible enough to handle multiple conditions at once.

Reflection on the Process

At first I wrote each filter separately and they did not work well together. If I cleared one filter it would ignore the others and reset everything. Moving to a single applyFilters() function forced me to treat the tickets array as data that flows through a series of conditions. That made the behavior much easier to reason about. I also learned to think more carefully about how and when I call renderTickets() so I am not duplicating work. Overall, this part of the project helped me connect what I learned about arrays, filtering, and sorting to a real UI with actual user controls.
