Database Artifact

Artifact Description

For this milestone, I worked on the database and backend for my Full Stack Ticketing System. This enhancement replaces the temporary localStorage setup with a real MySQL database and a Node.js/Express API. The database stores tickets and comments, and the backend handles all communication between the frontend and MySQL.

Why I Chose This Artifact

I selected this artifact for my ePortfolio because it shows that I can move beyond frontend work and build the backend structure that makes the application fully functional. Adding a database and API demonstrates my ability to design structured data, build backend routes, and connect all parts of a full stack application.

Key Enhancements

For this database focused milestone, I implemented the following:

  • Created the MySQL database ticket_system along with tables for tickets and comments.
  • Added a relational structure where comments reference the ticket they belong to using a foreign key.
  • Set up a Node.js backend using Express and mysql2 with a connection pool.
  • Built secure API routes for creating, retrieving, updating, and deleting tickets.
  • Updated the frontend so it communicates with the backend API instead of using localStorage.
  • Introduced data validation and parameterized queries to prevent SQL injection.

It also touches the security mindset by ensuring queries are parameterized and the backend is responsible for safely handling user input. While building this enhancement, I learned how the backend and frontend depend on each other and how important consistent data flow is. The biggest challenge was getting MySQL installed and configured correctly on Windows, especially resolving connection errors and authentication issues. Once the database connection was working, everything started coming together and the project felt like a real full stack application instead of a frontend demo.
